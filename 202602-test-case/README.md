## 活動簡介

本次分享探討 Anthropic 官方推出的 AI CLI 工具 Claude Code，以及如何在開發流程中有效整合它。Claude Code 讓開發者能直接在終端機操作 AI，支援 VS Code、JetBrains 等 IDE 擴充套件，並且具備讀檔、寫檔、執行指令、搜尋程式碼等強大功能。本次活動旨在介紹 Claude Code 的多元應用場景，從理解陌生程式碼、實作功能、除錯，到撰寫文件與自動化重複性任務，讓開發者能更高效地運用 AI 輔助軟體開發。

## 什麼是 Claude Code？

Claude Code 是由 Anthropic 官方開發的 AI 命令行介面 (CLI) 工具，旨在將強大的人工智慧能力無縫整合到開發者的日常工作流程中。它不僅能直接在終端機環境下執行，還提供 VS Code、JetBrains 等主流 IDE 的擴充套件，讓 AI 成為開發者身邊的得力助手。Claude Code 的核心能力包括讀取及修改檔案、執行指令、以及高效地搜尋程式碼，為開發者帶來前所未有的便利性。

---

## 適合拿來做什麼？

Claude Code 能夠應用於軟體開發的各個環節，有效提升開發效率與程式碼品質。
### 理解陌生 Codebase
當面對一個全新的專案或不熟悉的模組時，Claude Code 可以幫助快速理解程式碼的邏輯與功能，例如回答「這個 function 在幹嘛？」這類問題。
### 實作與除錯
您可以指示 Claude Code 進行功能的實作、修正程式碼中的錯誤 (bug)，或是撰寫必要的測試案例。
### 程式碼重構 (Refactor)
對於老舊或難以維護的程式碼，Claude Code 可以協助進行重構，優化結構與效能。
### 文件撰寫
在提交程式碼時，Claude Code 可以協助撰寫 commit message 或 Pull Request (PR) 的描述，確保資訊的完整性與清晰度。
### 自動化重複性任務
許多開發過程中會遇到重複性的任務，Claude Code 能夠自動化這些流程，節省寶貴的時間。

## 推薦工作流程：Plan Mode

在直接對程式碼進行修改之前，強烈建議先讓 Claude 進行規劃。這有助於確保 AI 理解任務的全貌，並產出一個符合預期的執行方案。
### 啟動 Plan Mode
您可以透過指令 ` /plan 幫我把這個 API 從 REST 改成 GraphQL` 來啟動 Plan Mode。
### 規劃與確認
Claude 會先探索目前的 codebase，並與您確認需求。在產出詳細的執行計畫後，請您仔細審閱並確認無誤，再授權 Claude 執行修改。
### 避免方向錯誤
這種方式能有效避免在開發過程中，因為方向錯誤而浪費時間進行無效的修改。

## CLAUDE.md：給 AI 的說明書

為了讓 Claude Code 更深入地理解您的專案，並遵循既有的規範，建議您在專案的根目錄建立一個 `CLAUDE.md` 檔案。
### 定義專案規範
在這個檔案中，您可以明確定義專案的開發慣例，例如：
- **程式語言要求**：例如「使用 TypeScript，避免 any」。
- **測試框架**：例如「測試框架：Vitest」。
- **Commit 訊息格式**：例如「commit 訊息格式：feat/fix/docs: 描述」。
- **程式碼撰寫準則**：例如「不要自動新增 console.log」。
### 自動載入與應用
每次您與 Claude Code 進行對話時，它都會自動載入此檔案，並在執行的過程中遵循其中定義的規範，省去您重複說明專案細節的麻煩。

## 善用 Subagents

對於較為複雜的任務，您可以考慮將任務拆解，並分配給不同的 Subagents 協同處理，以提高效率與精確度。
### Explore Agent
此 Agent 擅長快速探索 codebase，幫助您初步了解專案的結構與內容。
### Plan Agent
此 Agent 則專注於設計詳細的實作方案與執行步驟。
### 整合與執行
您可以在主流程中整合不同 Agent 的產出，並最終執行修改。
### 適用場景
這種方式特別適合用於進行大型的程式碼重構，或是需要深入理解多個檔案之間相互關係的任務。

## 費用控制技巧

在使用 Claude Code 的過程中，控制 token 使用量對於管理費用至關重要。以下提供幾個實用的技巧：
### 壓縮對話歷史
使用 `/compact` 指令可以壓縮對話歷史，減少 token 的消耗。
### 新對話與短 Context
任務完成後，建議開啟新的對話，避免 context 累積過長，維持 AI 的效率與準確性。
### 明確限定範圍
在請求修改時，請盡可能明確告知 Claude Code 的操作範圍，例如「只改這個 function，不要動其他地方」，以避免不必要的 token 浪費。
### 善用 Plan Mode
Plan Mode 有助於在修改前就確定好方向，減少 AI 進行多次嘗試而產生的額外 token 消耗。

## 實際踩過的坑

在實際應用 Claude Code 的過程中，我們也遇到了一些挑戰，並總結出一些經驗教訓，希望能幫助大家避開雷區：
### 1. 不要讓 AI 一次改太多
一次請求 AI 進行大量的程式碼修改，可能會導致難以追蹤與驗證。建議將任務拆解成小步驟，並在每一步都進行確認。
### 2. 敏感檔案要加進 .gitignore
務必將包含敏感資訊的檔案（如 `.env`、credentials 等）加入 `.gitignore`，以免 AI 在處理過程中不小心將其暴露。
### 3. Auto-approve 要小心
雖然自動批准可以加快流程，但對於大動作的修改，還是建議人工仔細確認後再批准，以防萬一。
### 4. Context 太長會變笨
當對話的 context 累積過長時，AI 的表現可能會下降。適時開啟新的對話，可以幫助 AI 保持較佳的狀態。

## 總結

Claude Code 最適合的定位是成為一個「有能力但需要你掌舵」的工程師。
### 你的角色
您負責提供清晰的方向、進行判斷、以及最終的確認。
### Claude 的角色
Claude 則負責執行探索、實作細節、以及處理繁瑣的程式碼撰寫工作。
### 最佳實踐
將 Claude Code 視為一個優秀的 Pair Programming 對象，而不是一個完全自動化的工具，這樣才能最大化其效益。
