## 活動簡介

本次活動旨在介紹 Anthropic 官方推出的 AI CLI 工具 Claude Code，以及分享其在開發流程中的推薦使用方式。Claude Code 能夠直接在終端機操作，整合進開發者的日常工作，支援 VS Code、JetBrains 等 IDE 擴充套件，並具備讀檔、寫檔、執行指令、搜尋程式碼等能力。透過本次分享，希望能幫助開發者更有效地利用 AI 工具來提升程式碼的理解、實作、重構與自動化效率。

Claude Code 不僅能幫助開發者理解陌生程式碼、實作新功能、修復 bug，還能協助撰寫 commit message、PR 描述，甚至自動化重複性的任務。本次活動將聚焦於 Claude Code 的實際應用，提供具體的建議與工作流程，讓與會者能將其有效融入現有的開發習慣中。

## Claude Code 核心功能與應用場景

Claude Code 是 Anthropic 專為開發者設計的命令行介面 (CLI) 工具，它提供了一個強大的 AI 助手，可以直接在終端機環境中進行互動。其核心能力包括：

-   **程式碼理解**：快速解釋特定函式的功能、程式碼片段的用意，讓你迅速掌握不熟悉的 codebase。
-   **程式碼生成與修改**：根據指令實作新功能、修復 bug、撰寫測試案例，或是對現有程式碼進行重構 (Refactor)。
-   **自動化任務**：例如，協助撰寫 commit message、產生 PR 描述，或是自動化其他重複性的開發流程。
-   **IDE 整合**：支援 VS Code、JetBrains 等主流 IDE 的擴充套件，讓 AI 的能力無縫接軌開發環境。

Claude Code 特別適合用於：
### 理解陌生 Codebase
當你接手一個新的專案或不熟悉的模組時，可以直接詢問 Claude Code：「這個 function 在幹嘛？」來加速理解過程。

### 實作與除錯
你可以請 Claude Code 協助實作特定的功能、修正發現的 bug，或是為程式碼撰寫單元測試，節省你手動撰寫的時間。

### 程式碼重構
對於老舊或效率不佳的程式碼，可以請 Claude Code 協助進行重構，使其更符合現代的程式碼規範或設計模式。

### 文件撰寫
撰寫 commit message 和 PR 描述往往是耗時但重要的工作，Claude Code 可以根據你的程式碼變更，生成結構化的描述，幫助你節省時間並提高文件品質。

### 自動化重複性任務
如同本次活動的 README 生成工具，Claude Code 也能夠自動化處理許多重複性的開發任務，釋放開發者精力。

## 推薦工作流程：Plan Mode

在進行程式碼修改或實作前，強烈建議優先使用 Claude Code 的 `Plan Mode`。這個模式能引導 AI 先進行規劃與探索，而非直接著手修改程式碼。

### 為什麼要使用 Plan Mode？
直接讓 AI 改寫程式碼，有時可能會導致方向錯誤，或是改到一半才發現設計上的問題，進而浪費時間。`Plan Mode` 透過讓 AI 先理解需求、探索 codebase，並產出一個詳細的實作計畫，來大幅降低這種風險。

### 如何使用 Plan Mode？
只需在對話中加入 `/plan` 指令，並描述你的目標。例如：
```
/plan 幫我把這個 API 從 REST 改成 GraphQL
```
Claude Code 會先與你互動，了解更細節的需求，然後提出一個詳細的執行步驟。在你確認計畫無誤後，AI 才會開始執行修改。這個過程能確保你和 AI 在同一個頻道上，減少後續返工的可能性。

## CLAUDE.md：專案說明書

為了讓 Claude Code 更了解你的專案、團隊的開發規範與偏好，你可以在專案的根目錄建立一個名為 `CLAUDE.md` 的檔案。這個檔案將作為 AI 的「說明書」，在每次與 Claude Code 對話時自動載入。

### CLAUDE.md 的內容範例
```markdown
# 專案規範

- **程式語言**：主要使用 TypeScript，嚴格避免使用 `any` 型別。
- **測試框架**：專案採用 Vitest 作為測試框架。
- **Commit 訊息格式**：遵循 Conventional Commits 規範，格式為 `feat/fix/docs: 簡短描述`。
- **程式碼風格**：禁止自動在程式碼中新增 `console.log` 語句。
```

### 善用 CLAUDE.md 的好處
-   **一致性**：確保 AI 在產生程式碼或提供建議時，能夠遵循專案的既有規範，維持程式碼風格與品質的一致性。
-   **效率提升**：無需每次對話都重複解釋專案的細節，AI 能自動參考 `CLAUDE.md`，直接進入實質的任務處理。
-   **減少誤解**：透過清晰的規範，能減少 AI 對於專案預期的誤解，進而提高產出結果的準確性。

## 善用 Subagents 處理複雜任務

面對複雜或龐大的開發任務時，直接丟給單一 AI Agent 處理可能會效果不佳。Claude Code 支援「Subagents」的概念，可以將複雜任務拆解成多個子任務，並分配給不同專長的 Agent 並行處理。

### Subagents 的應用
-   **Explore Agent**：專門負責快速探索 codebase，理解檔案結構、函式依賴等。
-   **Plan Agent**：負責根據 Explore Agent 的發現，設計出詳細的實作方案。
-   **主流程 Agent**：整合其他 Agent 的結果，並執行實際的程式碼修改。

### 適用場景
這種拆解任務的方式特別適合用於大型的程式碼重構、需要跨越多個檔案的重構，或是需要深入理解整個系統架構的任務。透過 Subagents，可以更有條理地解決複雜問題，並確保每個環節都得到妥善處理。

## 費用控制技巧

使用 AI 工具時，了解其計費方式並採取適當的節約技巧，能幫助你有效控制成本。Claude Code 的費用通常與使用的 Token 數量有關。

### 實用技巧
1.  **壓縮對話歷史**：使用 `/compact` 指令來壓縮對話歷史，減少不必要的 Token 消耗。
2.  **開啟新對話**：當一個任務告一段落，或是對話歷史過長影響 AI 表現時，開啟一個新的對話，避免持續累積過長的 Context。
3.  **明確指定範圍**：在發出指令時，盡可能明確告知 AI 你希望修改的範圍，例如：「請只修改 `user.js` 中的 `getUserById` 函式，不要動其他地方。」
4.  **善用 Plan Mode**：透過 Plan Mode 先規劃，可以避免 AI 進行多次無效的嘗試，從而節省 Token。

## 實際踩過的坑與建議

在使用 Claude Code 的過程中，我們也遇到了一些挑戰，並從中學習到一些寶貴的經驗。

1.  **不要讓 AI 一次改太多**：大型的程式碼修改，最好拆解成小步驟，每完成一個步驟就進行確認。這樣更容易發現問題並進行修正。
2.  **敏感檔案要加進 `.gitignore`**：確保你的 `.gitignore` 檔案中包含了所有敏感資訊的檔案，例如 `.env`、帳號密碼、API 金鑰等，避免 AI 在不經意間存取或透露這些資訊。
3.  **Auto-approve 要小心**：如果你的工作流程中有自動批准變更的機制，使用 AI 進行重大修改時要特別謹慎。建議對於重要的修改，還是要人工審核。
4.  **Context 太長會變笨**：AI 的表現會受到對話 Context 長度的影響。當 Context 過長時，AI 的記憶力可能會下降，或是變得不那麼聰明。適時開啟新的對話是個好方法。

## 總結：Claude Code 的定位

Claude Code 最適合被定位為一個「有能力但需要你掌舵」的工程師夥伴。

-   **你的角色**：負責設定方向、進行判斷、最終確認修改。
-   **Claude Code 的角色**：負責探索程式碼、進行實作、處理細節。

將 Claude Code 視為一個極具生產力的 Pair Programming 對象，而不是一個全自動化的工具。透過有效的溝通與指導，你可以極大化 AI 在開發流程中的效益，同時確保程式碼品質與專案方向的正確性。
