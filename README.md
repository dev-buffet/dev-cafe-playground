# 開發者 Cafe 活動紀錄

我們是一群在台灣科技研討會結識的夥伴，自疫情期間，我們所在的城市技術社群與聚會逐漸減少。希望能保持討論和交流的熱度，因此我們決定發起聚會。 在這裏會舉辦各種形式的活動，議題大致上包括但不限於前、後端、UI/UX、ML、軟體開發等，希望大家能在這裡找到並盡情享受自己感興趣的事物，因此取名為`開發者 Buffet`。

`開發者 Café`是我們目前主要的實體活動，舉辦於每月的最後一個周二晚上 19:00 - 21:00，在高雄的咖啡廳(目前主要定居在 [Second Space](https://www.secondspace.dev/zh/))與最多 10 位左右的開發者們一起輕鬆聊聊天，歡迎任何主題與討論。

有鑒於先前的活動沒有留下紀錄，我們特地建立這個 Repo 來存儲開發者 Cafe 的活動紀錄，包括投影片、照片、相關筆記與連結等。

## 如何上傳活動紀錄

如果你是這次的講者，請依照以下步驟上傳素材：

1. **建立目錄**：在 repo 根目錄新增一個以年月命名的資料夾，例如 `202603/` 或 `202603-主題名稱/`
2. **放入素材**：將投影片（建議命名為 `Slides.md`）或其他相關檔案放進去
3. **開 PR**：從你的分支開一個 PR 合併回 `main`，**PR 描述請照範本填寫**（包含分享者、活動連結與簡介）
4. **等待自動生成**：PR 開啟後，GitHub Actions 會自動根據你的素材與 PR 描述產生 `README.md`
5. **確認後合併**：確認產生的活動紀錄無誤後即可合併

> 投影片建議使用 Markdown 格式（`.md`），方便 AI 讀取內容。若為 PDF 或其他格式，請在 PR 描述中補充說明。
